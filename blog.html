1Q. Discuss the scope of var, let, and const

Scope:

Scope determines the accessibility of Variables. JavaScript have 3 types of Scope:

1.Block Scope
2.Function Scope
3.Global Scope

1.Block Scope: ES6 introduce two important JavaScript Keyword:

1.let
2.const

These two Keyword provide Block Scope in JavaScript.Variables declared with var keyword can't have block scope.

Variables declared inside curly brackets {} block can't be accessed from outside the block if it is declared with let/const.

2.Local Scope/Function Scope: Each function create a new Scope. Variables defined inside a function can't be accessible from outside the function. Variables declared with var, let, const are quite similar when declared inside a function.

var, let , const all have function scope.

3.Global Scope: A Variable declared outside a block and function scope , becomes global. All scripts and functions on a web page can access it.

2Q. Tell us the use cases of null and undefined

Use Cases for null:

1.Explicitly Empty: When you want to explicitly indicate that a variable or object property should be empty or have no value.

2.Initializing: Sometimes, you may set a variable to null as an initial value and later assign it an actual value when needed.

3.Resetting:You might set a variable to null to reset or clear it.

Use Cases for undefined:

1.Uninitialized Variables:When a variable is declared but not assigned a value, it's automatically undefined.

2.Function Return:When a function doesn't have a return statement or returns nothing explicitly, it returns undefined.

3.Missing Object Properties: When you try to access an object property that doesn't exist, you'll get undefined.

3Q. What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a design style for creating web services. It uses standard HTTP methods like GET (retrieve), POST (create), PUT (update), and DELETE (remove) to perform operations on resources, which are identified by URLs. REST APIs are stateless and have a uniform interface, making them simple and scalable. They're widely used for building web services that can be accessed by various clients.
